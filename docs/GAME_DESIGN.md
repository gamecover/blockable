Blockable 기획안 v0.4

프로젝트 목적
- NHN 2026 해커톤 참여를 목적으로 제작하는 웹 기반 게임 프로토타입이다.
- 공식 페이지: https://nan2026.nhn.com/

작성 기준
- 현재까지 논의된 세계관, 지도, 전투, 블록, 보상, 이벤트 및 UI 구성을 통합한 기획 초안이다.
- v0.3 본문과 문서 내 '현재 정리 과정에서 확정한 기준'을 우선 적용한다.
- 편의상 사용한 '레고'라는 표현은 실제 게임 제작 시 '블록'으로 통일한다.
- 최근 대화에서 직접 확정한 UI 구성은 본문에 반영한다.
- 문서 마지막에 적힌 추가 내용은 기본적으로 제안 사항이며, 기존 본문을 자동으로 대체하지 않는다.
- 지도는 프로토타입 검증을 위해 템플릿 기반 제약형 무작위 생성 방식을 우선 적용한다. 난이도별 층·노드 수와 세부 확률은 아직 확정되지 않았다.
- 비용, 확률, 피해량 등 본문에 남아 있는 숫자는 별도 표기가 없는 한 프로토타입 검증용 임시값이다.

3. 주요 게임 구조

3-1. 지도 용어

던전(Dungeon)
- 플레이어가 공략하는 전체 지역이다.
- 하나의 던전은 여러 개의 층으로 구성될 수 있다.
- 예: 잿빛 용광로

층(Floor)
- 던전을 구성하는 세로 단계(Vertical Level)다.
- 플레이어가 한 층의 보스 노드를 완료하면 다음 층으로 이동한다.

노드 위치(Node Step)
- 한 층 안에서의 가로 진행 순서(Horizontal Level)다.
- 예: '잿빛 용광로 1-3'에서 1은 1층, 3은 해당 층의 세 번째 노드 위치를 의미한다.

노드(Node)
- 전투나 이벤트가 발생하는 실제 개별 공간이다.
- 같은 노드 위치에 여러 개의 선택 가능한 노드가 존재할 수 있다.
- 같은 위치의 노드는 내부적으로 1-3-A, 1-3-B, 1-3-C처럼 고유 ID를 부여해 구분한다.
- 플레이어 화면에서는 필요한 경우 모두 1-3으로 표시한다.

지도 계층
던전
└─ 여러 층
   └─ 여러 노드 위치
      └─ 하나 이상의 전투·이벤트 노드

3-2. 기본 지도 구조
- 지도는 Slay the Spire처럼 노드와 노드가 연결된 분기형 구조다.
- 던전에 입장하면 지도를 무작위로 생성한다.
- 진행 방향은 지도 왼쪽 끝인 9시 방향에서 오른쪽 방향이다.
- 각 층에는 시작 노드와 보스 노드가 하나씩 존재한다.
- 시작 노드를 완료하면 연결된 다음 노드를 선택할 수 있다.
- 노드를 완료하면 지도 화면으로 돌아온다.
- 현재 노드와 직접 연결된 다음 노드만 선택할 수 있다.
- 경로는 중간에 분기되거나 다시 합류할 수 있다.
- 완료한 노드는 완료 상태로 표시한다.
- 층 보스를 처치하면 다음 층으로 이동한다.
- 마지막 층의 보스를 처치하면 던전을 클리어한다.
- 플레이어가 사망하면 현재 런을 종료한다.

3-3. 지도 진행 흐름
1) 던전 입장
2) 지도 생성
3) 층 시작 노드 진행
4) 연결된 다음 노드 선택
5) 전투 또는 이벤트 진행
6) 보상 획득 및 덱 정비
7) 지도 화면으로 복귀
8) 다음 노드 선택
9) 층 보스 진행
10) 다음 층 이동
11) 마지막 층 보스 처치
12) 던전 클리어

3-4. 노드 및 인카운터 종류

시작 노드
- 해당 층의 출발 지점이며 각 층에 하나만 존재한다.

일반 전투 노드
- 블록 배치 퍼즐을 이용해 몬스터와 싸운다.
- 몬스터를 처치하면 골드와 블록 보상을 획득한다.

이벤트 노드
- 블록 추가, 삭제, 색상·능력 변경, 형태 변형, 상점, 보물 상자 및 일반 선택 이벤트가 발생할 수 있다.
- 선택을 완료하거나 스킵하면 종료된다.
- 미믹이나 공허 괴물이 등장하면 전투 인카운터로 전환될 수 있다.

보스 노드
- 해당 층의 마지막 노드다.
- 일반 경로는 최종적으로 보스 노드에 연결된다.
- 일반 층에는 층 보스가, 마지막 층에는 던전 최종 보스가 등장한다.

3-5. 난이도와 지도 규모
- 던전 난이도에 따라 층 수, 층별 노드 위치 수 및 위치별 노드 수가 달라질 수 있다.
- 이 방향성은 프로토타입에서 검증하며 정확한 수치는 아직 확정하지 않는다.

생성 변수
- 층 수: 던전 전체 길이
- 층별 노드 위치 수: 한 층에서 진행하는 가로 단계 수
- 위치별 노드 수: 같은 진행 위치에서 선택 가능한 공간 수
- 분기 횟수: 경로 선택이 발생하는 빈도
- 합류 횟수: 나뉜 경로가 다시 합쳐지는 빈도
- 노드 종류 비율: 전투·이벤트 등의 배치 비율
- 정보 공개 범위: 앞으로 확인할 수 있는 노드 범위
- 전투 등급: 일반·강화·엘리트·보스 전투의 배치

프로토타입용 임시 기준
- 쉬움: 1층, 층별 7개 노드 위치, 위치별 1~2개 노드. 짧고 단순한 경로.
- 보통: 2층, 층별 8~10개 노드 위치, 위치별 1~3개 노드. 분기와 합류가 반복되는 경로.
- 어려움: 3층, 층별 10~12개 노드 위치, 위치별 2~3개 노드. 긴 경로와 위험한 선택 증가.
- 최고 난이도: 3층 이상, 층별 12개 이상 노드 위치, 위치별 2~4개 노드. 경로별 위험과 보상 차이 확대.
- 위 수치는 프로토타입 검증용이며 확정 밸런스가 아니다.

3-6. 한 층의 기본 구성
- 한 층은 '입구 → 첫 번째 분기 → 분기 경로 유지 → 일부 경로 합류 → 두 번째 분기 → 보스 전 준비 → 층 보스' 흐름을 기본 골격으로 사용한다.
- 노드 위치가 10개인 층은 다음 구성을 예시로 사용할 수 있다.
  - 1: 층 시작 노드
  - 2~3: 첫 번째 분기
  - 4: 일부 경로 합류
  - 5~7: 주요 경로 선택 구간
  - 8: 보스 전 합류
  - 9: 준비 또는 위험·보상 선택
  - 10: 층 보스
- 모든 층을 동일하게 만들지 않고 여러 구조 템플릿 중 하나를 선택해 변형한다.

3-7. 지도 생성 방식
- 완전 무작위 생성이 아니라 미리 정의한 층 구조에 제약된 무작위를 적용한다.

생성 순서
1) 던전 난이도를 기준으로 전체 층 수를 결정한다.
2) 각 층의 노드 위치 수와 위치별 최대 노드 수를 결정한다.
3) 시작 노드와 보스 노드를 먼저 생성한다.
4) 시작 노드에서 보스까지 이어지는 기본 경로를 2~3개 생성한다.
5) 기본 경로 사이에 분기와 합류를 추가한다.
6) 연결되지 않은 노드와 막다른 경로가 있는지 검사한다.
7) 지도 연결 구조가 완성된 후 노드 종류를 배정한다.
8) 모든 선택 가능한 경로가 보스까지 도달하는지 검사한다.
9) 생성에 사용한 랜덤 시드를 저장한다.

지도 구조 템플릿 후보
- 초반에 넓게 분기한 뒤 합류하는 구조
- 초반에는 한 길이고 후반에 분기하는 구조
- 두 경로 사이에서 선택이 반복되는 구조
- 안전하지만 긴 경로와 위험하지만 짧은 경로
- 보상 노드를 중심으로 우회하는 구조

3-8. 지도 생성 제약
- 모든 시작 경로는 보스까지 연결되어야 한다.
- 막다른 경로를 생성하지 않는다.
- 이전 노드로 돌아갈 수 없다.
- 완료한 노드에 다시 진입할 수 없다.
- 현재 위치와 연결된 다음 노드만 활성화한다.
- 분기된 경로는 최소 두 개의 노드 위치 동안 유지한다.
- 분기 직후 합류하는 의미 없는 선택은 만들지 않는다.
- 하나의 노드에서 연결되는 다음 노드는 최대 2~3개로 제한한다.
- 모든 최종 경로는 보스 노드에 연결한다.
- 전투 또는 이벤트가 지나치게 연속되지 않도록 제한한다.
- 위험한 경로에는 그에 맞는 보상을 배치한다.
- 서로 다른 경로가 완전히 같은 구성을 갖지 않도록 한다.
- 마지막 층의 보스는 일반 층 보스와 구분한다.

3-9. 지도 표시 규칙
- 현재 위치는 다른 노드와 명확하게 구분한다.
- 완료한 노드는 완료 상태로 표시한다.
- 선택 가능한 다음 노드만 활성화한다.
- 연결되지 않은 노드는 선택할 수 없도록 한다.
- 노드 사이의 연결 경로를 화면에 표시한다.
- 노드 종류는 아이콘이나 색상으로 구분한다.
- 같은 노드 위치에 여러 선택지가 있으면 위·중간·아래에 나누어 배치한다.
- 상단 진행 정보는 '잿빛 용광로 1-3' 또는 '잿빛 용광로 / 1층 · 3/10' 형식으로 표시한다.
- 기존의 '4/10F' 표시는 새 용어와 맞지 않으므로 사용하지 않는다.

3-10. 지도 저장 및 이어하기
- 현재 던전
- 현재 층
- 현재 노드 ID
- 지도 생성 랜덤 시드
- 생성된 전체 지도 구조
- 완료한 노드
- 선택 가능한 다음 노드
- 현재 진행 중인 인카운터
- 플레이어 체력과 골드
- 현재 덱 구성

자동 저장 시점 후보
- 노드 선택 직후
- 전투 종료 후
- 이벤트 종료 후
- 보상 선택 후
- 다음 층 진입 시
- 메인 화면으로 이동할 때

3-11. 지도 시스템 적용 상태

프로토타입에 적용
- 던전 → 층 → 노드 위치 → 실제 노드 구조
- 왼쪽에서 오른쪽으로 진행하는 분기형 지도
- 층마다 시작 노드와 보스 노드 배치
- 템플릿 기반 제약형 무작위 생성
- 경로 분기와 합류
- 이전 노드 이동 및 완료 노드 재진입 금지
- 모든 경로의 보스 연결 보장
- 난이도에 따라 층 수와 노드 수를 변경할 수 있는 구조

아직 확정되지 않은 항목
- 각 난이도의 정확한 층 수
- 층별 정확한 노드 위치 수
- 위치별 최대 노드 수
- 노드 종류별 등장 확률
- 전투·이벤트 연속 등장 제한 횟수
- 엘리트·상점·회복 노드 추가 여부
- 경로별 위험과 보상의 구체적인 계산 공식
- 보스 직전 노드의 고정 규칙
- 플레이어에게 공개할 미래 노드 범위


4. 블록 및 덱 시스템

4-1. 덱의 표현
- 게임에서 덱은 블록이 들어 있는 '주머니'로 표현한다.
- 가장 처음 전투를 시작할 때 총 13개의 블록을 가진다.
- 12개는 고정된 기본 블록으로 제공하고, 나머지 1개는 플레이어가 선택해 추가한다.
- 게임 시작 시 정한 기본 덱과 이후 전투·이벤트에서 변경된 덱 구성을 현재 런 동안 유지한다.
- 각 전투에서는 덱의 드로우 순서만 무작위로 섞는다.

4-2. 블록의 구성
- 블록은 여러 개의 정사각형 칸이 서로 붙어 있는 형태다.
- 일반 블록은 주로 3칸 또는 4칸으로 구성한다.
- 강하거나 희귀한 블록은 최대 3x3 범위 안에서 더 복잡한 형태를 가질 수 있다.
- 블록마다 모양, 색상, 능력, 수치, 희귀도 또는 등급을 가질 수 있다.

4-3. 기본 블록 종류

I 블록
- 형태: 정사각형 3개가 일자로 연결된 모양

O 블록
- 형태: 2x2 정사각형

L 블록
- 형태: 정사각형 3개가 ㄱ자로 연결된 모양

4-4. 모양과 색상
- 블록의 모양은 퍼즐 판에 배치되는 형태를 결정한다.
- 블록의 색상은 블록이 발동하는 능력 종류를 결정한다.
- 기본 블록은 무색이다.
- 현재 초안에서 무색 블록의 공격력은 10으로 두되, 확정 밸런스 수치는 아니다.
- 색상 변경 이벤트를 유지할 경우, 게임 진행 중 블록의 모양은 유지하면서 색상과 능력을 변경할 수 있다.

4-5. 특수 블록 후보
- U자 블록
- 지그재그 블록
- 십자형 블록
- 공격과 방어를 동시에 제공하는 복합 블록
- 프로토타입에서는 I, O, L 세 종류를 먼저 구현하고 특수 블록은 이후 확장한다.


5. 전투 시스템

5-1. 전투 개요
- 전투에서는 테트리미노처럼 여러 정사각형이 붙어 있는 블록을 퍼즐 판에 배치한다.
- 퍼즐 판은 칠교놀이처럼 일정한 외곽 형태를 가진다.
- 플레이어는 매 턴 무작위로 뽑은 블록을 배치해 공격, 방어, 회복 효과를 발생시킨다.
- 적에게는 '약점'이 존재한다.
- 특정 모양을 완성하거나 판을 정확히 채우면 추가 효과가 발생한다.
- 약점을 포함한 모양을 형성하면 추가 효과가 발생한다.

5-2. 전투 화면 전체 구성

좌측 상단 플레이어 정보 HUD
- 수평 체력 바와 현재 체력/최대 체력 수치
- 체력 바 아래에 이로운 상태 이상 아이콘을 수평 배치
- 그 아래에 해로운 상태 이상 아이콘을 수평 배치
- 그 아래에 현재 골드와 남은 덱 블록 수 표시

화면 중앙
- 플레이어 퍼즐 판과 몬스터 퍼즐 판을 배치한다.
- 플레이어와 몬스터의 실제 이미지 또는 연출 영역은 별도로 구성할 수 있다.
- 이번 턴에 뽑은 블록을 표시한다.
- 남은 블록 확인 버튼, 버려진 블록 확인 버튼, 턴 종료 버튼을 배치한다.
- 블록을 집었을 때만 회전 조작 안내를 표시한다.
- 몬스터의 다음 행동과 예상 효과를 표시한다.

플레이어 퍼즐 판 아래 상태 표시
- 수평 체력 바와 체력 바 중앙의 현재 체력/최대 체력 수치
- 체력 바 아래에 이로운 상태 이상 아이콘을 수평 배치
- 그 아래에 해로운 상태 이상 아이콘을 수평 배치
- 그 아래에 이번 턴 배치한 블록 수/최대 배치 가능 수를 표시한다. 예: 2/3

몬스터 퍼즐 판 아래 상태 표시
- 수평 체력 바와 체력 바 중앙의 현재 체력/최대 체력 수치
- 체력 바 아래에 이로운 상태 이상 아이콘을 수평 배치
- 그 아래에 해로운 상태 이상 아이콘을 수평 배치
- 몬스터에게는 배치 블록 수를 표시하지 않는다.

우측 상단 정보 및 설정 HUD
- 현재 던전·층·노드 위치 표시. 예: 잿빛 용광로 1-3
- 지도 확인 버튼
- 설정 버튼: 톱니바퀴 아이콘 사용
- 메인 화면 이동 버튼: 문 아이콘 사용
- 자동 저장 상태 표시
- 메인 화면 이동 버튼은 오작동 방지를 위해 다른 버튼과 시각적으로 분리한다.
- 아이콘에 마우스를 올리면 기능 이름을 알려주는 툴팁을 표시한다.

기타 전투 정보
- 현재 턴
- 현재 노드 위치
- 필요한 경우 플레이 시간
- 도감 버튼은 기능 확정 후 추가한다.

5-3. 퍼즐 판 규칙
- 퍼즐 판은 블록을 놓을 수 있는 정사각형 칸들의 집합이다.
- 블록은 퍼즐 판 내부에만 배치할 수 있다.
- 블록끼리 겹칠 수 없다.
- 판 밖으로 나가는 배치는 허용하지 않는다.
- 칸마다 표시되던 대각선 또는 불필요한 내부 장식선은 제거한다.
- 격자와 외곽선만 사용해 가시성을 확보한다.

5-4. 퍼즐 판과 체력의 관계
- 블록 1칸을 체력 5에 대응시키는 현재안을 사용한다.
- 기본 체력 75일 경우 판은 총 15칸으로 구성된다.
- 체력이 줄어들 때마다 판의 칸이 무작위로 빠진다.
- 단, 가운데 최소 3x3 영역은 보장한다.
- 체력이 늘어날 때는 가장 최근에 빠진 칸을 먼저 복구한다.
- 빠진 칸을 모두 복구한 뒤 추가 칸이 필요하면 테두리를 따라 순차적으로 생성한다.
- 몬스터의 외형 또는 특징에 맞는 판이 존재할 수 있다.
- 몬스터 판은 추가 효과를 주거나 특정 기믹을 수행하는 데 사용될 수 있다.

5-5. 드로우와 덱 순환
1) 턴 시작 시 덱에서 블록 5개를 무작위로 뽑는다.
2) 플레이어는 뽑은 블록 중 기본적으로 최대 3개를 퍼즐 판에 배치한다.
3) 추가 효과가 있으면 최대 배치 가능 수가 늘어날 수 있다.
4) 턴 종료 전까지 사용하지 않은 블록은 버림 더미로 이동한다.
5) 이번 턴에 사용한 블록도 효과 처리 후 버림 더미로 이동한다.
6) 덱에 있는 블록을 모두 사용하면 버림 더미를 다시 덱으로 가져온다.
7) 덱을 무작위로 섞은 뒤 다시 블록을 뽑는다.

5-6. 남은 블록 확인
- 남은 블록 버튼은 덱에 있지만 아직 사용되지 않은 블록의 목록을 보여준다.
- 블록의 종류와 수량을 확인할 수 있다.
- 다음에 뽑힐 블록을 예측할 수 없도록 화면 표시 순서를 무작위로 섞는다.
- 실제 드로우 순서는 노출하지 않는다.

5-7. 버려진 블록 확인
- 버려진 블록 버튼은 현재 버림 더미에 있는 블록 목록을 보여준다.
- 지금까지 사용했거나 사용하지 못해 버린 블록을 확인할 수 있다.
- 버튼은 턴 종료 버튼의 왼쪽에 배치한다.

5-8. 블록 조작

블록 배치
- 블록을 마우스로 드래그해 퍼즐 판에 배치한다.
- 배치 가능한 위치에 들어가면 가능한 상태임을 시각적으로 표시한다.
- 판 밖으로 나가거나 다른 블록과 겹치면 배치할 수 없다.

블록 회전
- 블록을 드래그하는 도중 R 키를 누르면 오른쪽으로 90도 회전한다.
- R 키를 반복해서 누르면 90도 단위로 계속 회전한다.
- 정사각형처럼 회전해도 형태가 같은 블록은 동일하게 표시한다.

블록 회수
- 턴 종료 전에는 이미 배치한 블록을 다시 퍼즐 판에서 꺼낼 수 있다.
- 꺼낸 블록은 이번 턴의 손패로 돌아간다.
- 회수한 블록은 다른 위치에 다시 배치할 수 있다.
- 사용하지 않은 상태로 턴을 종료하면 버림 더미로 이동한다.
- 턴 종료 후에는 배치를 되돌릴 수 없다.

5-9. 기본 턴 흐름
0) 턴 시작
1) 몬스터 행동 결정 후 표시
2) 주머니에서 블록 5개 드로우
3) 매 턴 블록 회전 및 퍼즐 판 배치 가능 횟수 제공. 기본 최대 3회
4) 필요하면 배치한 블록 회수 및 재배치
5) 플레이어가 턴 종료 선택
6) 개별 블록 효과 계산
7) 특정 모양 및 세트 효과 계산
8) 계산 결과를 플레이어와 몬스터에 적용
9) 몬스터가 생존했다면 몬스터의 행동 및 결과 적용
10) 배치한 블록을 퍼즐 판에서 모두 제거
11) 사용한 블록과 사용하지 않은 블록을 버림 더미로 이동
12) 플레이어 방어도를 0으로 초기화
13) 다음 턴 시작

추후 개발 후보
- 도감에 등록된 형태가 현재 손패에 존재하면 해당 형태를 즉시 배치하는 기능

5-10. 퍼즐 판 초기화
- 턴 종료 시 그 턴에 배치한 블록을 모두 제거한다.
- 이전 턴의 블록 배치는 다음 턴에 남지 않는다.
- 다음 턴에는 빈 퍼즐 판에서 다시 배치한다.

5-11. 블록 기본 효과
- 무색 블록으로 시작해 이벤트 등으로 색을 입히거나 다시 뺄 수 있다.
- 색깔별로 고유한 능력이 지정되어 있다.
- 현재 초안에서는 배치한 무색 블록당 공격력 10을 적용한다.
- 특정 모양으로 조합하면 추가 효과가 발생한다.
- 색상별 효과와 최종 수치는 추후 밸런스 테스트로 확정한다.

5-12. 방어도
- 방어도는 몬스터의 공격 피해를 대신 받는다.
- 받은 피해가 방어도보다 크면 남은 피해가 플레이어 체력에 적용된다.
- 현재 턴에 획득한 방어도는 해당 턴의 몬스터 공격을 막을 수 있다.
- 방어도는 턴이 끝날 때마다 0으로 초기화한다.

예시
- 플레이어 방어도: 10
- 몬스터 공격력: 15
- 방어도가 피해 10을 막는다.
- 남은 피해 5가 플레이어 체력에 적용된다.

5-13. 특정 모양 완성 효과
- 여러 블록을 조합해 지정된 모양을 만들면 특수 효과가 발생한다.
- 완성한 모양에 포함된 블록에 추가 배율 또는 별도 효과를 적용한다.

5-14. 중복 세트 보너스
- 같은 턴에 같은 종류의 모양을 여러 개 완성하면 효과가 증가한다.
- 현재 예시: 2x4 또는 4x2 직사각형 세트를 2개 만들면 해당 세트에 포함된 모든 블록 효과가 2.5배로 증가한다.
- 세트 개수별 정확한 조건과 배율은 밸런스 테스트를 통해 결정한다.

5-15. 퍼즐 판 완성 보너스
- 퍼즐 판을 빈칸 없이 정확하게 채우면 추가 능력이 발동한다.
- 후보 효과는 추가 공격, 모든 공격 블록 효과 증가, 추가 방어도, 추가 골드, 다음 턴 추가 드로우 등이다.
- 판 완성 보너스는 일반 블록 효과 및 모양 세트 효과를 모두 계산한 후 계산한다.

5-16. 효과 계산 순서
1) 개별 블록 효과
2) 특정 모양 세트 효과
3) 동일 세트 중복 보너스
4) 몬스터 판 완성 보너스
5) 퍼즐 판 완성 보너스
6) 계산된 공격·방어·회복 결과 적용

5-17. 몬스터 행동
- 몬스터는 턴마다 공격 또는 특수 행동을 한다.
- 몬스터의 다음 행동은 플레이어 행동 전에 결정해 화면에 표시한다.
- 플레이어의 블록 효과를 처리한 뒤 몬스터가 생존했을 경우에 행동한다.
- 플레이어의 공격으로 몬스터가 사망했다면 해당 턴의 몬스터 행동은 실행하지 않는다.
- 상태 이상 시스템을 추가할 경우 특정 칸 봉인, 특정 색상 약화, 배치 방향 제한, 방해 블록 추가 등의 행동을 사용할 수 있다.

5-18. 전투 종료 조건

승리
- 몬스터의 체력이 0 이하가 되는 순간 전투가 종료된다.

패배
- 플레이어의 체력이 0 이하가 되는 순간 전투가 종료된다.

후퇴
- 보스방을 제외한 일반 몬스터 전투의 후퇴 가능 여부와 처리 방식은 아직 결정되지 않았다.


6. 전투 결과 및 보상

6-1. 골드 보상
- 전투에서 승리하면 몬스터에 따라 정해진 범위 내에서 기본 골드를 받는다.
- 구체적인 획득량과 계산 공식은 아직 확정되지 않았다.

6-2. 블록 보상
- 일반 전투 종료 후 무작위 블록 3개를 제시한다.
- 블록 선택지에는 무색 및 유색 블록이 포함될 수 있다.
- 플레이어는 블록 1개를 선택해 덱에 추가할 수 있다.
- 필요하지 않다면 아무것도 선택하지 않고 스킵할 수 있다.
- 보상 처리가 끝나면 지도 화면으로 복귀한다.

6-3. 인카운터별 결과

일반 전투
- 성공 조건: 몬스터 처치
- 보상: 골드, 블록 3개 중 1개 선택
- 종료 후: 지도 복귀

보스 전투
- 성공 조건: 보스 처치
- 보상: 던전 클리어 보상. 다회차 플레이가 생기면 추후 확장
- 종료 후: 게임 클리어

일반 이벤트
- 성공 조건: 선택 완료 또는 스킵
- 보상: 이벤트에 따라 다름
- 종료 후: 지도 복귀

이벤트 전투
- 성공 조건: 미믹 또는 공허 괴물 처치
- 보상: 특수 보상 또는 일반 전투 보상
- 종료 후: 지도 복귀
- 프로토타입 이후 개발한다.

상점 이벤트
- 완료 조건: 구매 후 나가기, 또는 판매와 변형 중 하나를 수행한 뒤 나가기
- 변형은 플레이어가 지정한 블록의 칸 하나를 3x3 범위 안의 다른 위치로 옮기는 방식이다.
- 구매하거나 정비한 덱은 현재 런 동안 유지한다.
- 종료 후 지도 화면으로 복귀한다.


7. 이벤트 시스템

7-1. 이벤트 분류
- 이벤트 인카운터는 일반 이벤트와 레어 이벤트로 나뉜다.
- 현재 초안의 등장 비율은 일반 이벤트 70%, 레어 이벤트 30%다.
- 이 비율은 지도·경제 공식과 함께 추후 검증한다.

7-2. 일반 이벤트

상점
- 골드로 블록을 구매할 수 있다.
- 보유한 블록을 판매할 수 있다.
- 보유한 블록을 변형시킬 수 있다.
- 판매와 변형은 둘 중 하나만 수행할 수 있다.
- 구매 목록과 판매 목록은 리롤할 수 없다.
- 이용하지 않고 나갈 수 있다.

무료 블록 추가
- 무작위 블록 5개를 제시한다.
- 그중 1개를 선택해 덱에 추가할 수 있다.
- 스킵할 수 있다.
- 리롤할 수 없다.
- 최대 1회 선택할 수 있다.

무료 블록 삭제
- 현재 덱에 있는 블록 중 1개를 제거할 수 있다.
- 스킵할 수 있다.
- 최대 1회 가능하다.

무료 모양 변경 또는 색 변경
- 현재 덱에 있는 블록 1개의 모양이나 색깔을 변경할 수 있다.
- 스킵할 수 있다.
- 최대 1회 가능하다.

유료 블록 추가
- 무작위 블록 3개를 제시한다.
- 무색, 유색 및 특수 블록이 등장할 수 있다.
- 골드를 지불하고 그중 1개를 덱에 추가할 수 있다.
- 스킵할 수 있다.
- 목록을 리롤할 수 있다.
- 기존 초안의 임시 리롤 비용은 최초 25골드이며, 리롤할 때마다 최초 비용만큼 증가한다.

유료 블록 삭제
- 골드를 지불해 덱의 블록 1개를 제거할 수 있다.
- 스킵할 수 있다.
- 기존 초안의 임시 비용은 최초 50골드이며, 제거할 때마다 최초 비용만큼 증가한다.

유료 모양 변경 또는 색 변경
- 골드를 지불해 현재 덱에 있는 블록 1개의 모양이나 색깔을 변경할 수 있다.
- 스킵할 수 있다.
- 모양 변경은 블록의 모든 칸을 이동시킬 수 있다.
- 기존 초안의 임시 비용은 최초 50골드이며, 변경할 때마다 최초 비용만큼 증가한다.
- 최대 2회 가능하다.

보물 상자
- 플레이어는 상자를 열거나 지나갈 수 있다.
- 상자를 열면 골드, 미믹, 공허의 상자 또는 광대 상자 등의 결과가 발생할 수 있다.
- 골드 보상, 결과별 확률 및 비용은 현재 확정 공식이 없으므로 추후 결정한다.
- 미믹은 즉시 전투로 전환되는 후보 콘텐츠다.
- 공허의 상자는 골드 또는 체력을 대가로 요구하는 후보 콘텐츠다.
- 광대 상자는 지울 수 없는 저주 블록을 추가하는 후보 콘텐츠다.

7-3. 레어 이벤트

생명의 샘
- 플레이어의 최대 체력을 늘릴 수 있다.
- 현재 초안의 임시 증가량은 최대 체력 25다.
- 최대 체력이 늘어난 만큼 현재 체력도 회복한다.

7-4. 이벤트 종료
- 이벤트 선택을 완료하거나 스킵하면 인카운터가 종료된다.
- 미믹이나 공허 괴물 전투로 전환되면 해당 몬스터를 처치해야 완료된다.
- 이벤트가 끝나면 지도 화면으로 복귀하고 다음 분기 노드를 선택한다.


8. 재화 시스템

8-1. 기본 재화
- 기본 재화는 골드이다.

8-2. 골드 획득처
- 일반 전투 승리
- 보스 전투 승리
- 보물 상자
- 기타 이벤트 결과

8-3. 골드 사용처
- 블록 추가

- 블록 색상 변경
- 기타 이벤트 비용 지불
- 아이템 구매. 추후 개발

8-4. 경제 밸런스의 현재 상태
- 골드 획득량과 소비 비용의 최종 기준 및 계산 공식은 아직 없다.
- 본문에 남아 있는 25골드, 50골드 등의 수치는 기존 초안의 프로토타입용 임시값이다.
- 일반 전투 보상, 상점 가격, 리롤·삭제·변경 비용은 플레이 테스트 후 조정한다.
- 마지막 추가 메모에 제시된 100골드, 150골드, 150~300골드 등의 값은 본문에 반영하지 않은 제안값이다.


9. 추가 개발 후보

9-1. 유물 시스템. 우선순위 1
- 한 번의 게임이 끝날 때까지 지속되는 패시브 효과를 제공한다.

예시
- 첫 번째로 배치한 빨간 블록의 효과 증가
- 퍼즐 판을 완성하면 추가 골드 획득
- 매 전투 첫 턴에 블록 1개 추가 드로우

9-2. 특수 아이템
- 일회성 또는 제한 횟수로 사용하는 능동형 아이템이다.

예시
- 현재 퍼즐 판을 특정 모양으로 변경
- 블록 1개 재드로우
- 배치한 블록 1개 회수
- 특정 칸의 잠금 해제

9-3. 점수 및 랭킹
- 게임을 클리어하거나 종료했을 때 최종 점수를 계산한다.
- 점수를 이용해 랭킹을 보여줄 수 있다.

점수 요소 후보
- 도달한 층
- 처치한 몬스터 수
- 남은 체력
- 보유 골드
- 오버킬 피해량
- 퍼즐 판 완성 횟수
- 특정 모양 완성 횟수
- 클리어에 사용한 턴 수

9-4. 상태 이상. 우선순위 2
- 플레이어와 몬스터에게 일시적인 효과를 부여한다.

예시
- 화상
- 중독
- 취약
- 공격력 감소
- 방어도 획득량 감소
- 특정 색상 봉인
- 퍼즐 판의 특정 칸 봉인

9-5. 튜토리얼
- 튜토리얼의 필요성은 있으나 구체적인 구성, 진행 순서 및 표시 공식은 아직 확정되지 않았다.
- 첫 전투에서 필요한 순간에 단계별로 안내하는 방식은 제안 상태로 유지한다.
- 드래그, R 키 회전, 유효 위치, 블록 회수, 최대 3개 배치, 색상별 효과, 몬스터 행동 예고, 턴 종료, 덱·버림 더미 확인 등이 안내 후보이다.


10. 공통 화면 및 UI 동작

10-1. 메인 화면
- NEW GAME
- CONTINUE
- 설정
- 종료 또는 메인 화면에서 필요한 기타 메뉴
- CONTINUE가 실제로 동작하기 위한 저장 규칙은 아직 최종 확정되지 않았다.

10-2. 설정 및 지도 버튼
- 메인 화면을 제외한 화면의 우측 상단에는 현재 던전·층·노드 위치, 지도, 설정, 메인 화면 이동 버튼을 배치한다.
- 설정에는 효과음·배경음 음량과 전체 화면 전환 기능을 포함하는 안을 우선 검토한다.
- 메인 화면 이동 시 확인 창을 표시한다.
- 브라우저 기본 alert보다 게임 내부 모달을 사용한다.
- 설정 창이 열린 동안 배경 화면의 입력을 차단한다.
- 전투 중에는 블록 드래그, 회전 및 턴 종료 입력도 차단한다.

10-3. 자동 저장 표시
- 우측 상단에 작은 자동 저장 상태 표시를 둔다.
- 저장 중, 저장 완료, 저장 실패 상태를 색상 또는 짧은 애니메이션으로 구분한다.
- 실제 저장 시점과 저장 데이터 범위는 부록의 제안 규칙을 검토한 뒤 확정한다.

10-4. UI 표현 방향
- 저채도 금속 프레임과 제한적인 불빛을 사용해 대장간·던전 분위기를 표현한다.
- 플레이어 상태 표시는 차가운 청색 계열, 몬스터 상태 표시는 달궈진 적색 계열을 사용할 수 있다.
- 상태 아이콘은 금속 표찰이나 공구처럼 체력 바 아래에 매달린 형태를 사용할 수 있다.
- 상태 아이콘이 많아질 경우 최대 표시 개수와 +N 형태의 초과 표시 규칙이 필요하다.
- 상태 추가·제거 시에만 짧게 움직이고 지속적인 흔들림은 사용하지 않는다.


11. 확정 사항과 추가 결정 사항

11-1. 현재 확정한 기준
- 기본 덱은 13개다. 12개 고정 제공 후 1개를 선택해 추가한다.
- 지도는 던전 → 층 → 노드 위치 → 실제 노드의 계층으로 구성한다.
- 지도는 왼쪽에서 오른쪽으로 진행하는 분기형 구조로 생성한다.
- 각 층에는 시작 노드와 보스 노드가 하나씩 존재한다.
- 프로토타입에서는 템플릿 기반 제약형 무작위 생성을 적용한다.
- 이전 노드로 돌아가거나 완료한 노드에 다시 진입할 수 없다.
- 모든 경로는 해당 층의 보스 노드까지 연결되어야 한다.
- 블록 3개 중 하나를 선택하는 보상은 일반적으로 전투 승리 보상이다.
- 이벤트 인카운터는 각 이벤트에 정해진 별도 결과를 적용한다.
- 턴 종료 시 이번 턴에 놓은 블록을 전부 제거한다.
- 다음 턴은 빈 퍼즐 판에서 시작한다.
- 턴 종료 전에는 배치한 블록을 회수하고 재배치할 수 있다.
- 한 턴에는 기본적으로 블록을 최대 3개 배치한다.
- 플레이어의 효과를 먼저 처리하고, 생존한 몬스터가 행동한다.
- 방어도는 턴 종료 시 0으로 초기화한다.
- 마지막 층에는 일반 층 보스와 구분되는 던전 최종 보스가 존재한다.
- 좌측 상단, 중앙의 양측 판 아래, 우측 상단의 전투 UI 구조를 사용한다.
- 플레이어 판 아래에는 이번 턴 배치 블록 수를 표시한다.

11-2. 지도와 진행. 추후 결정
- 각 난이도의 정확한 층 수
- 층별 정확한 노드 위치 수와 위치별 최대 노드 수
- 노드 종류별 등장 확률과 연속 등장 제한 횟수
- 엘리트·상점·회복 노드 추가 여부
- 경로별 위험과 보상의 구체적인 계산 공식
- 보스 직전 노드의 고정 규칙
- 플레이어에게 공개할 미래 노드 범위
- 일반 몬스터, 엘리트 몬스터, 보스 구성
- 던전 최종 보스와 최종 클리어 연출

11-3. 덱과 블록. 추후 결정
- 기본 덱 13개의 정확한 종류별 개수
- 덱이 지나치게 작아지는 것을 막기 위한 최소 블록 수
- 블록 희귀도와 획득 확률
- 블록 효과를 모양과 영구 고정할지, 색상 변경으로 바꿀 수 있게 할지
- 색상 변경 시 공격력, 방어도, 회복량을 정하는 기준
- 복합 효과 블록의 구체적인 규칙
- 블록 효과를 블록 단위로 계산할지 칸 단위로 계산할지

11-4. 퍼즐과 조작. 추후 결정
- 퍼즐 판의 기본 크기와 체력 연동 방식의 최종 확정
- 몬스터 형태 기반과 층 난이도 기반 중 어느 방식으로 판을 정할지
- 블록을 드래그하지 않는 상태에서도 회전할 수 있는지
- 모바일 또는 터치 환경에서 회전을 어떻게 조작할지
- 배치 가능한 위치를 어떤 방식으로 표시할지
- 완성 모양을 판정할 때 좌우 대칭, 높낮이 및 회전을 어떻게 인정할지

11-5. 전투 규칙. 추후 결정
- 플레이어 기본 능력치의 종류와 최종 계산식
- 특정 모양의 종류와 기본 보너스
- 중복 세트 개수별 효과 배율
- 여러 세트가 같은 블록을 공유할 수 있는지
- 퍼즐 판 완성 보너스의 정확한 효과
- 공격, 방어, 회복 수치에 세트 배율을 동일하게 적용할지
- 체력, 공격력, 방어도 등 기본 밸런스 수치
- 오버킬 골드 계산식과 상한
- 일반 전투 후퇴 가능 여부와 처리 방식
- 몬스터 데이터 구조와 행동 선택 공식

11-6. 이벤트와 경제. 추후 결정
- 일반 이벤트와 레어 이벤트 내부의 개별 등장 확률
- 상점 판매 가격과 구매 가격
- 유료 추가, 삭제, 변경 및 리롤의 최종 비용
- 미믹과 공허 괴물의 난이도 및 보상
- 공허 괴물 전투 승리 시 특수 보상 여부
- 이벤트 전투에도 일반 전투의 블록 선택 보상을 지급할지
- 골드 획득량과 소비 비용의 전체 경제 공식

11-7. 저장 및 이어하기. 추후 결정
- 자동 저장 시점
- 저장할 런 데이터의 정확한 범위
- 전투 중 저장 허용 여부
- 전투 중 종료 후 CONTINUE 시 재개 위치
- 패배 시 현재 런 저장 데이터 삭제 시점
- 설정값과 프롤로그 열람 여부의 별도 저장 여부


12. 한 문장 요약

난이도에 따라 여러 층과 노드로 생성되는 분기형 던전에서 전투와 이벤트를 선택하고, 매 턴 블록 5개를 뽑아 그중 최대 3개를 퍼즐 판에 배치해 공격·방어·회복 및 모양 완성 효과를 발생시키며 덱을 강화해 최종 보스를 처치하는 로그라이크 블록 배치 게임이다.


부록 A. 마지막 추가 메모의 제안 사항

주의
- 이 부록은 확정 규칙이 아니다.
- 기존 본문과 충돌할 경우 기존 본문을 우선한다.
- 이후 사용자가 별도로 승인한 내용만 본문 규칙으로 옮긴다.

A-1. 공통 설정 메뉴 제안
- 메인 화면을 제외한 모든 화면에 공통 설정 버튼 표시
- 설정 열기
- 메인 화면으로 이동
- 효과음 및 배경음 음량 조절
- 전체 화면 전환
- 설정 창이 열린 동안 현재 화면의 게임 입력 차단
- 메인 화면 이동 시 확인 창 표시
- 메인 화면 이동 전 자동 저장

A-2. 플레이어 능력치 제안
- 최대 체력: 플레이어가 가질 수 있는 최대 HP
- 현재 체력: 0 이하가 되면 패배
- 공격력 또는 공격력 보정: 공격 블록 효과에 적용
- 방어도: 현재 턴에 받는 피해를 먼저 흡수
- 회복량 또는 회복량 보정: 회복 블록 효과에 적용
- 골드: 상점과 유료 이벤트에서 사용

A-3. 피해 처리 공식 제안
1) 공격 피해 계산
2) 방어도만큼 피해 흡수
3) 남은 피해를 체력에서 차감
4) 체력이 0 이하인지 확인
5) 사망 또는 다음 행동 진행

A-4. 세부 턴 흐름 제안
1) 플레이어 턴 시작
2) 덱에서 블록 5개 뽑기
3) 몬스터가 다음 행동 예고
4) 플레이어가 블록을 최대 3개 배치
5) 턴 종료
6) 배치된 블록의 효과 계산
7) 모양·세트·퍼즐 완성 보너스 계산
8) 몬스터 피해 및 사망 확인
9) 생존한 몬스터가 행동
10) 플레이어 피해 및 사망 확인
11) 사용·미사용 블록을 버림 더미로 이동
12) 퍼즐 판에 놓인 블록 제거
13) 플레이어 방어도 초기화
14) 다음 턴 시작

A-5. 몬스터 전투 데이터 객체화 제안
- 이름 및 식별 ID
- 최대 체력과 현재 체력
- 행동 목록과 행동별 수치
- 행동 선택 조건
- 다음 행동 예고 정보
- 연속 행동 제한
- 상태 이상
- 처치 시 골드 범위
- 특수 보상
- 등장 가능한 층
- 퍼즐 판 형태

예시: 불의 정령
- 체력: 50
- 불꽃 공격: 피해 10
- 화염 강화: 다음 공격 피해 +5
- 강한 불꽃: 피해 18, 연속 사용 불가
- 기본 골드: 80~100

A-6. 저장 및 이어하기 제안

자동 저장 시점 후보
- 노드 선택 직후
- 전투 종료 후
- 이벤트 종료 후
- 보상 선택 후
- 다음 층 진입 시
- 메인 화면으로 이동할 때

저장 항목 후보
- 현재 던전, 층, 노드 ID
- 지도 생성 랜덤 시드
- 현재 체력
- 보유 골드
- 현재 덱
- 지도 구조
- 완료한 노드
- 다음에 진입할 인카운터
- 프롤로그 열람 여부
- 설정값

전투 중 저장 제안
- 반복 재시도 악용을 방지하기 위해 전투 도중의 상태는 저장하지 않는다.
- 전투 중 종료하면 해당 전투의 시작 상태부터 다시 진행하는 안을 검토한다.
- 이 규칙을 적용하면 현재 손패, 버림 더미, 드로우 순서, 배치 블록, 몬스터 현재 체력, 현재 턴 및 방어도는 저장하지 않는다.

A-7. 전투 종료 및 실패 처리 제안

승리
1) 골드 계산
2) 오버킬 보너스 계산
3) 블록 보상 3개 제시
4) 하나 선택하거나 건너뛰기
5) 자동 저장
6) 지도 화면으로 복귀

패배
1) 게임 오버 화면 표시
2) 이번 플레이 결과 표시
3) 현재 런의 이어하기 데이터 삭제
4) 메인 화면으로 이동

A-8. 지도 생성 제안
- 기존 A-8의 제안 중 지도 구조와 생성 제약은 본문 3장에 프로토타입 적용 기준으로 반영했다.
- 난이도별 정확한 수치, 노드 배치 확률, 위험·보상 계산식은 아직 확정되지 않았다.

A-9. 경제 밸런스 제안값
- 일반 전투 평균 보상: 100골드
- 강한 전투 평균 보상: 150골드
- 블록 구매: 150~300골드
- 블록 삭제: 최초 200골드
- 유료 변경: 최초 150골드
- 리롤: 최초 50골드
- 위 수치는 확정값이 아니며 기존 본문의 임시 비용을 대체하지 않는다.

A-10. 튜토리얼 제안
- 첫 전투에서 필요한 순간에 한 단계씩 표시
- 블록 드래그
- R 키 회전
- 유효한 칸에만 배치 가능
- 이미 놓은 블록 회수
- 한 턴 최대 3개 사용
- 블록 색상별 효과
- 몬스터 행동 예고
- 턴 종료
- 덱과 버림 더미 확인
- 구체적인 진행 순서, 표시 방식 및 재열람 규칙은 아직 없음
