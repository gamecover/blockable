기획안 v0.3

작성 기준
- 현재까지 논의된 세계관, 화면, 지도, 전투, 블록, 보상 및 이벤트 시스템을 통합한 기획안이다.
- 이 문서는 게임의 동작과 콘텐츠를 정의하며, 기술 구조와 코드 작성 방식은 별도의 CODEX_DEVELOPMENT_GUIDE.md를 따른다.
- 같은 항목에 서로 다른 내용이 있으면 v0.3에 새로 작성된 구체적인 수치를 우선한다.
- 문서에서 '추후', '이후', '나중에', '추가 개발', '후보'라는 의미로 작성된 내용은 삭제하지 않고 기록한다.
- 위 표현이 붙은 내용은 '향후 적용 항목'이며 현재 버전에는 구현하지 않는다.
- 현재 적용 항목과 향후 적용 항목이 충돌하면 현재 적용 항목을 우선한다.

표기 규칙
- [현재 적용]: 지금 개발할 기능과 규칙이다.
- [향후 적용]: 문서에는 유지하지만 현재 개발에서는 구현하지 않는다.
- [미정]: 아직 수치나 동작을 확정하지 않은 항목이다.
- [Codex 재량]: 마지막 항목의 기준에 따라 Codex가 최소 범위에서 결정할 수 있다.

1-3. 핵심 플레이

- 전투 또는 이벤트 인카운터를 진행한다.

- 플레이어가 선택한 경로를 기준으로 총 10층을 진행해 마지막 보스를 처치한다.

2. 세계관 및 프롤로그

2-1. 기본 설정
세계 제일의 대장장이인 당신.

그러나 당신의 실력을 비웃기라도 하는 듯, 당신이 만든 무기로 출정을 떠난 용사들은 하나같이 던전을 정복하지 못하고 패퇴를 반복하고 있었다.

답답함을 참지 못한 당신은 자신의 '도구 주머니'를 들고 직접 던전을 정복하기로 한다.

당신은 과연 던전을 정복할 수 있을까?

2-2. 프롤로그 표현
- 프롤로그는 일러스트 1장과 이야기 문구로 구성한다.
- 게임을 처음 실행했을 때 한 번만 자동으로 표시한다.
- 프롤로그를 본 기록은 브라우저의 저장 데이터에 유지한다.
- 저장 데이터가 없거나 초기화된 경우 다시 최초 실행으로 취급할 수 있다.
- 프롤로그가 끝나면 지도 화면으로 이동한다.

4. 지도 및 인카운터 구조

4-1. 지도 시스템
- 지도는 Slay the Spire처럼 노드와 노드가 연결된 분기형 구조다.
- 던전을 시작하면 지도를 무작위로 생성한다.
- 진행 방향은 지도 왼쪽 끝인 9시 방향에서 오른쪽 방향이다.
- 1층에는 시작 노드 1개가 존재한다.
- 시작 노드를 완료한 뒤 다음 층부터 무작위 개수의 노드를 생성한다.
- 선택한 노드의 목표를 달성하면 지도가 다시 열린다.
- 플레이어는 현재 노드와 연결된 다음 분기 노드 중 하나를 선택한다.
- 같은 과정을 반복해 마지막 보스 노드까지 이동한다.
- 완료한 노드는 완료 상태로 표시한다.
- 현재 위치와 직접 연결된 노드만 선택할 수 있다.

4-2. 층과 노드
- 한 던전은 플레이어가 진행하는 경로를 기준으로 총 10층이다.
- 1층은 시작 노드다.
- 2층부터 9층까지는 일반 전투 또는 이벤트 노드다.
- 10층은 보스 노드다.
- 각 층에는 선택하지 않는 노드를 포함해 여러 노드가 존재할 수 있다.
- 플레이어는 각 층에서 노드 하나를 선택하므로 한 번의 플레이에서 총 10개 노드를 거친다.
- 지도 전체에는 분기를 만들기 위해 10개보다 많은 노드가 생성된다.
- 노드 경로는 중간에 분기되거나 다시 합류할 수 있다.

4-3. 지도 구성 요소
- 시작 노드
- 일반 전투 노드
- 이벤트 노드
- 분기 경로
- 보스 노드

4-4. 인카운터 종류

전투 인카운터
- 블록 배치 퍼즐을 이용해 몬스터와 싸운다.
- 몬스터를 처치하면 성공한다.

이벤트 인카운터
- 블록 추가, 삭제, 변경, 상점, 보물 상자 등의 선택지를 진행한다.
- 선택을 완료하거나 스킵하면 종료된다.
- 이벤트가 전투로 전환되는 기능은 향후 적용 항목에 별도로 기록한다.


5. 블록 및 덱 시스템

5-1. 덱의 표현
- 게임에서 덱은 블록이 들어 있는 '도구 주머니'로 표현한다.
- 최초 전투 진입 시 블록 13개를 가지고 시작한다.
- 12개는 고정된 기본 블록으로 제공한다.
- 나머지 1개는 제시된 선택지 중 하나를 골라 추가한다.
- 선택한 기본 덱 구성은 이후 전투에도 유지된다.
- 전투 시작 시 덱의 구성은 바꾸지 않고 드로우 순서만 무작위로 섞는다.
- 전투와 이벤트를 통해 추가하거나 삭제한 블록은 이후 전투에도 유지된다.

5-2. 블록의 구성
- 블록은 여러 개의 정사각형 칸이 서로 붙어 있는 형태다.
- 일반 블록은 주로 3칸 또는 4칸으로 구성한다.
- 블록마다 모양, 색상, 능력 및 수치를 가질 수 있다.

5-3. 현재 기본 블록 종류

I 블록
- 정사각형 3개가 일자로 연결된 모양이다.

O 블록
- 정사각형 4개가 2x2 형태로 연결된 모양이다.

L 블록
- 정사각형 3개가 ㄱ자 형태로 연결된 모양이다.

5-4. 모양과 색상
- 블록의 모양은 퍼즐 판에 배치되는 형태를 결정한다.
- 블록의 색상은 블록이 발동하는 능력 종류를 결정한다.
- 기본 블록은 무색으로 시작한다.
- 무색 블록은 블록 1개를 정상적으로 배치할 때 기본 공격력 10을 발생시킨다.
- 게임 진행 중 이벤트를 통해 블록에 색을 입히거나 색을 제거할 수 있다.
- 색상별 고유 능력과 정확한 수치는 아직 미정이다.

5-5. 향후 특수 블록
[향후 적용 — 현재 구현 제외]
- U자 블록
- 지그재그 블록
- 십자형 블록
- 공격과 방어를 동시에 제공하는 복합 블록
- 최대 3x3 범위 안에서 구성되는 희귀 블록

현재 프로토타입에서는 I, O, L 세 종류를 우선 구현한다.


6. 전투 시스템

6-1. 전투 개요
- 여러 정사각형이 붙어 있는 블록을 퍼즐 판에 배치한다.
- 퍼즐 판은 일정한 외곽 형태를 가진다.
- 플레이어는 매 턴 무작위로 뽑은 블록을 이용해 공격, 방어 또는 기타 효과를 발생시킨다.
- 적에게는 '약점'이 존재한다.
- 특정 모양을 완성하거나 퍼즐 판을 정확히 채우면 추가 효과가 발생한다.
- 약점을 포함한 모양을 형성하면 추가 효과가 발생한다.

6-3. 퍼즐 판 기본 규칙
- 퍼즐 판은 블록을 놓을 수 있는 정사각형 칸들의 집합이다.
- 블록은 퍼즐 판 내부에만 배치할 수 있다.
- 블록끼리 겹칠 수 없다.
- 판 밖으로 나가는 배치는 허용하지 않는다.
- 불필요한 대각선 또는 내부 장식선을 제거한다.
- 격자와 외곽선만 사용해 가시성을 확보한다.

6-4. 플레이어 퍼즐 판과 체력
- 플레이어의 기본 최대 체력은 75다.
- 퍼즐 판의 블록 칸 1개는 체력 5에 대응한다.
- 기본 체력 75일 때 퍼즐 판은 총 15칸이다.
- 체력이 줄어 퍼즐 판의 칸이 감소할 때 제거할 칸은 무작위로 선택한다.
- 퍼즐 판 중앙의 최소 3x3 영역은 항상 보장한다.
- 체력이 회복되어 칸이 다시 생길 때 가장 최근에 제거된 칸부터 복구한다.
- 제거된 칸을 모두 복구한 뒤 최대 체력이 증가한 경우에는 기존 판의 테두리를 따라 순차적으로 칸을 생성한다.
- 체력과 퍼즐 판의 정확한 갱신 시점 및 체력이 5의 배수가 아닐 때의 처리는 Codex 재량 규칙을 따른다.

6-5. 몬스터 퍼즐 판
- 몬스터의 외형 또는 특징에 맞는 고유한 판이 존재할 수 있다.
- 몬스터 판은 추가 효과를 주거나 특정 기믹을 수행하는 데 사용한다.
- 몬스터 판의 구체적인 기믹이 명시되지 않은 경우 현재 전투에 필요한 가장 단순한 보너스 방식으로 구현할 수 있다.

6-6. 드로우와 덱 순환
1) 턴 시작 시 덱에서 블록 5개를 무작위로 뽑는다.
2) 플레이어는 뽑은 블록 중 기본적으로 최대 3개를 퍼즐 판에 배치할 수 있다.
3) 추가 효과를 통해 한 턴의 배치 가능 횟수를 늘릴 수 있다.
4) 턴 종료 전까지 사용하지 않은 블록은 버림 더미로 이동한다.
5) 이번 턴에 사용한 블록도 효과 처리 후 버림 더미로 이동한다.
6) 덱이 비면 버림 더미를 다시 덱으로 가져와 섞는다.
7) 이후 다시 블록을 뽑는다.

6-7. 남은 블록 확인
- 남은 블록 버튼은 덱에 있지만 아직 사용되지 않은 블록의 목록을 보여준다.
- 블록의 종류와 수량을 확인할 수 있다.
- 다음 드로우를 예측할 수 없도록 화면 표시 순서는 무작위로 섞는다.
- 실제 드로우 순서는 노출하지 않는다.
- 버튼의 정확한 위치는 UI 가시성을 해치지 않는 범위에서 Codex가 정할 수 있다.

6-8. 버려진 블록 확인
- 버려진 블록 버튼은 현재 버림 더미의 블록 목록을 보여준다.
- 사용했거나 사용하지 못해 버린 블록을 확인할 수 있다.
- 버튼은 턴 종료 버튼의 왼쪽에 배치하는 것을 기본으로 한다.

6-9. 블록 조작

블록 배치
- 블록을 마우스로 드래그해 퍼즐 판에 배치한다.
- 배치 가능한 위치에 들어가면 가능한 상태임을 시각적으로 표시한다.
- 판 밖으로 나가거나 다른 블록과 겹치면 배치할 수 없다.

블록 회전
- 블록을 드래그하는 도중 R 키를 누르면 오른쪽으로 90도 회전한다.
- R 키를 반복하면 90도 단위로 계속 회전한다.
- 회전은 배치 횟수를 소비하지 않는 것을 기본으로 한다.
- 정사각형처럼 회전해도 형태가 같은 블록은 동일하게 표시한다.

블록 회수
- 턴 종료 전에는 이미 배치한 블록을 퍼즐 판에서 다시 꺼낼 수 있다.
- 꺼낸 블록은 이번 턴의 손패로 돌아간다.
- 회수한 블록은 다른 위치에 다시 배치할 수 있다.
- 회수와 재배치를 배치 횟수로 다시 계산할지 여부는 Codex가 사용성에 맞게 결정한다.
- 사용하지 않은 상태로 턴을 종료하면 버림 더미로 이동한다.
- 턴 종료 후에는 배치를 되돌릴 수 없다.

6-10. 기본 턴 흐름
1) 턴 시작
2) 몬스터의 다음 행동 결정 및 표시
3) 주머니에서 블록 5개 드로우
4) 플레이어에게 기본 3회의 블록 배치 기회 제공
5) 플레이어가 블록을 회전, 배치, 회수 또는 재배치
6) 플레이어가 턴 종료 선택
7) 개별 블록 효과 계산
8) 특정 모양 및 세트 효과 계산
9) 몬스터 판 효과 계산
10) 플레이어 퍼즐 판 완성 효과 계산
11) 계산 결과를 플레이어와 몬스터에게 적용
12) 몬스터가 생존했다면 표시된 행동 실행
13) 몬스터 행동 결과 적용
14) 배치한 블록을 퍼즐 판에서 모두 제거
15) 사용한 블록과 사용하지 않은 블록을 버림 더미로 이동
16) 플레이어 방어도를 0으로 초기화
17) 다음 턴 시작

[향후 적용 — 현재 구현 제외]
- 도감에 등록된 형태가 손패에 존재할 경우 즉시 배치하는 기능

6-11. 턴 간 퍼즐 판 초기화
- 턴 종료 시 그 턴에 배치한 블록을 모두 제거한다.
- 이전 턴의 블록 배치는 다음 턴에 남지 않는다.
- 다음 턴에는 빈 퍼즐 판에서 다시 배치한다.

6-12. 블록 기본 효과
- 기본 무색 블록은 배치한 블록 1개당 공격력 10을 발생시킨다.
- 효과 계산 단위는 블록을 구성하는 칸이 아니라 배치한 블록 1개다.
- 색상별로 별도의 고유 능력을 지정할 수 있다.
- 특정 모양을 완성하면 추가 효과가 발생한다.

6-13. 방어도
- 방어도는 몬스터의 공격 피해를 대신 받는다.
- 받은 피해가 방어도보다 크면 남은 피해가 플레이어 체력에 적용된다.
- 현재 턴에 획득한 방어도는 해당 턴의 몬스터 공격을 막을 수 있다.
- 몬스터의 행동이 끝난 뒤 턴 종료 처리에서 방어도를 0으로 초기화한다.

예시
- 플레이어 방어도: 10
- 몬스터 공격력: 15
- 방어도가 피해 10을 막는다.
- 남은 피해 5가 플레이어 체력에 적용된다.

6-14. 특정 모양 완성 효과
- 여러 블록을 조합해 지정된 모양을 만들면 특수 효과가 발생한다.
- 완성한 모양에 포함된 블록에 추가 배율 또는 별도 효과를 적용한다.
- 완성 모양의 구체적인 종류와 기본 보너스는 미정이다.

6-15. 중복 세트 보너스
- 같은 턴에 같은 종류의 모양을 여러 개 완성하면 효과가 증가한다.
- 예: 2x4 또는 4x2 직사각형 세트를 2개 만들면 해당 세트에 포함된 모든 블록의 효과가 2.5배로 증가한다.
- 세트 개수별 정확한 조건과 배율은 밸런스 테스트를 통해 결정한다.

6-16. 퍼즐 판 완성 보너스
- 플레이어 퍼즐 판을 빈칸 없이 정확하게 채우면 추가 능력이 발동한다.
- 후보 효과에는 추가 공격, 공격 블록 효과 증가, 추가 방어도, 추가 골드, 다음 턴 추가 드로우 등이 있다.
- 정확한 효과가 확정되기 전에는 현재 전투에 필요한 단순한 추가 공격 보너스를 사용할 수 있다.
- 퍼즐 판 완성 보너스는 일반 블록 효과와 모양 세트 효과를 계산한 뒤 계산한다.

6-17. 효과 계산 순서
1) 개별 블록 효과
2) 특정 모양 세트 효과
3) 동일 세트 중복 보너스
4) 몬스터 판 완성 또는 기믹 보너스
5) 플레이어 퍼즐 판 완성 보너스
6) 최종 공격, 방어 및 기타 효과 적용

6-18. 몬스터 행동
- 몬스터는 턴마다 공격 또는 특수 행동을 한다.
- 플레이어의 블록 효과를 처리한 뒤 몬스터가 생존한 경우에만 행동한다.
- 플레이어의 공격으로 몬스터가 사망하면 해당 턴의 몬스터 행동은 실행하지 않는다.
- 몬스터 행동은 별도 개발 지침에 정의된 XState 기반 규칙형 및 확률형 State Machine을 따른다.

[향후 적용 — 현재 구현 제외]
- 상태 이상을 이용한 특정 칸 봉인
- 특정 색상 약화
- 배치 방향 제한
- 방해 블록 추가

6-19. 전투 종료 조건

승리
- 몬스터 체력이 0 이하가 되는 즉시 전투가 종료된다.

패배
- 플레이어 체력이 0 이하가 되는 즉시 전투가 종료된다.
- 패배 후에는 게임오버 화면으로 이동하는 것을 기본으로 한다.


7. 전투 결과 및 보상

7-1. 골드 보상
- 전투에서 승리하면 몬스터에 따라 정해진 범위 내에서 기본 골드를 받는다.
- 정확한 몬스터별 골드 범위는 미정이다.

7-2. 블록 보상

- 각 선택지는 현재 출현 가능한 무색 및 유색 블록 전체를 기준으로 동일한 1/n 확률로 선택하는 것을 기본으로 한다.
- 플레이어는 블록 1개를 선택해 덱에 추가할 수 있다.
- 필요하지 않다면 아무것도 선택하지 않고 스킵할 수 있다.
- 보상 처리가 끝나면 지도 화면으로 복귀한다.

일반 이벤트
- 성공 조건: 선택 완료 또는 스킵
- 보상: 이벤트에 따라 다름
- 종료 후: 지도 복귀

[향후 적용 — 현재 구현 제외] 이벤트 전투
- 성공 조건: 미믹 또는 공허 괴물 처치

8. 이벤트 시스템

8-1. 이벤트 분류
- 이벤트 인카운터는 일반 이벤트와 레어 이벤트로 나뉜다.
- 일반 이벤트 등장 확률은 70%다.
- 레어 이벤트 등장 확률은 30%다.

8-2. 일반 이벤트

상점
- 골드로 블록을 구매할 수 있다.
- 보유한 블록을 판매할 수 있다.
- 보유한 블록을 변형할 수 있다.
- 한 번의 상점 방문에서 판매와 변형 중 하나만 이용할 수 있다.
- 구매는 판매 또는 변형 선택과 별개로 이용할 수 있다.
- 구매 목록은 리롤할 수 없다.
- 아무것도 이용하지 않고 나갈 수 있다.
- 변형은 선택한 블록의 칸을 3x3 판 안에서 이동해 모양을 바꾸는 방식이다.
- 변형 후 블록의 모든 칸은 서로 연결되어야 하며, 칸 수는 유지하는 것을 기본으로 한다.

무료 블록 추가
- 무작위 블록 5개를 제시한다.
- 각 선택지는 출현 가능한 무색 및 유색 블록 전체를 기준으로 동일한 1/n 확률로 선택하는 것을 기본으로 한다.
- 그중 1개를 선택해 덱에 추가할 수 있다.
- 스킵할 수 있다.
- 리롤할 수 없다.
- 최대 1회 선택할 수 있다.

무료 블록 삭제
- 현재 덱에 있는 블록 중 1개를 제거할 수 있다.
- 스킵할 수 있다.
- 최대 1회 가능하다.

무료 모양 변경 또는 색상 변경
- 현재 덱에 있는 블록 1개의 모양 또는 색상을 변경할 수 있다.
- 한 번의 이벤트에서 모양 변경과 색상 변경 중 하나를 선택한다.
- 스킵할 수 있다.
- 최대 1회 가능하다.

유료 블록 추가
- 무작위 블록 3개를 제시한다.
- 각 선택지는 출현 가능한 무색 및 유색 블록 전체를 기준으로 동일한 1/n 확률로 선택하는 것을 기본으로 한다.
- [향후 적용] 특수 블록이 10% 확률로 출현할 수 있다.
- 골드를 지불하고 그중 1개를 덱에 추가할 수 있다.
- 스킵할 수 있다.
- 목록을 리롤할 수 있다.
- 기본 리롤 비용은 25골드다.
- 리롤할 때마다 비용이 25골드씩 증가한다.
- 최대 리롤 횟수는 미정이다.

유료 블록 삭제
- 골드를 지불해 덱의 블록 1개를 제거할 수 있다.
- 스킵할 수 있다.
- 기본 비용은 50골드다.
- 제거할 때마다 비용이 50골드씩 증가한다.
- 최대 제거 횟수는 미정이다.

유료 모양 변경 또는 색상 변경
- 골드를 지불해 현재 덱에 있는 블록 1개의 모양 또는 색상을 변경할 수 있다.
- 한 번의 변경에서 모양 변경과 색상 변경 중 하나를 선택한다.
- 스킵할 수 있다.
- 모양 변경 시 블록의 모든 칸을 3x3 범위 안에서 이동할 수 있다.
- 변경 후 블록의 모든 칸은 서로 연결되어야 하며 칸 수는 유지한다.
- 기본 비용은 50골드다.
- 변경할 때마다 비용이 50골드씩 증가한다.
- 최대 2회 가능하다.

보물 상자
- 플레이어는 상자를 열거나 지나갈 수 있다.
- 현재 적용되는 결과 확률은 다음과 같다.

골드 상자: 95%
- 골드 상자가 선택되면 95% 확률로 150골드를 받는다.
- 골드 상자가 선택되면 5% 확률로 기본 보상의 2배인 300골드를 받는다.

공허의 상자: 5%
- 현재 보유 골드의 절반 또는 남은 체력의 25% 중 하나를 선택해 바친다.
- 상자 내부에 블랙홀이 보이는 느낌으로 연출한다.

광대 상자: 5%
- [향후 적용 — 현재 구현 제외]
- 제거할 수 없는 저주 블록을 덱에 추가한다.
- 삐에로 인형이 튀어나오는 느낌으로 연출한다.
- 광대 상자를 구현하는 향후 버전에서는 최상위 확률을 골드 상자 90%, 공허의 상자 5%, 광대 상자 5%로 변경한다.

미믹
- [향후 적용 — 현재 구현 제외]
- 보물 상자에서 등장해 즉시 전투를 시작한다.
- 현재 등장 확률은 정해지지 않았으므로 현재 보물 상자 확률표에 포함하지 않는다.

8-3. 레어 이벤트

생명의 샘
- 플레이어의 최대 체력을 25 증가시킨다.
- 증가한 최대 체력만큼 현재 체력도 회복한다.
- 최대 체력 증가에 맞춰 플레이어 퍼즐 판의 칸도 추가한다.

8-4. 이벤트 종료
- 이벤트 선택을 완료하거나 스킵하면 인카운터가 종료된다.
- 이벤트가 끝나면 지도 화면으로 복귀하고 다음 분기 노드를 선택한다.

[향후 적용 — 현재 구현 제외]
- 미믹 또는 공허 괴물 전투로 전환되는 이벤트
- 이벤트 전투가 발생하면 해당 몬스터를 처치해야 이벤트가 완료된다.

10. 현재 확정된 핵심 기준

10-1. 기본 덱
- 최초 전투 시작 시 블록 13개를 가진다.
- 12개는 고정 블록이며 1개는 선택해서 추가한다.
- 게임 중 변경된 덱 구성은 이후 전투에 유지된다.
- 각 전투에서는 드로우 순서만 무작위로 섞는다.

10-2. 지도
- 시작 노드는 1개다.
- 플레이어가 선택한 한 경로가 총 10개 노드다.
- 지도 전체에는 분기 때문에 10개보다 많은 노드가 존재한다.

10-3. 블록 보상
- 일반 전투 승리 후 블록 3개 중 하나를 선택한다.
- 블록 보상은 스킵할 수 있다.
- 이벤트는 각 이벤트에 정해진 결과를 적용한다.

10-4. 턴 규칙
- 매 턴 블록 5개를 뽑는다.
- 한 턴에 기본적으로 최대 3개를 배치한다.
- 사용 여부와 관계없이 턴이 끝나면 이번 손패의 블록은 버림 더미로 이동한다.
- 턴 종료 시 퍼즐 판의 배치 블록을 모두 제거한다.
- 다음 턴은 빈 퍼즐 판에서 시작한다.
- 몬스터 행동이 끝난 뒤 방어도를 0으로 초기화한다.

10-5. 기본 전투 수치
- 플레이어 기본 최대 체력은 75다.
- 체력 5당 퍼즐 판 1칸이 대응한다.
- 기본 무색 블록은 블록 1개당 공격력 10이다.
- 중앙의 최소 3x3 퍼즐 영역은 항상 유지한다.


11. 향후 적용 항목 — 현재 구현 제외

이 절의 모든 내용은 문서에 보존하지만 현재 개발 범위에는 포함하지 않는다.

11-1. 유물 시스템 — 향후 우선순위 1
- 한 번의 게임이 끝날 때까지 지속되는 패시브 효과를 제공한다.

예시
- 첫 번째로 배치한 빨간 블록의 효과 증가
- 퍼즐 판을 완성하면 추가 골드 획득
- 매 전투 첫 턴에 블록 1개 추가 드로우

11-2. 특수 아이템
- 일회성 또는 제한 횟수로 사용하는 능동형 아이템이다.

예시
- 현재 퍼즐 판을 특정 모양으로 변경
- 블록 1개 재드로우
- 배치한 블록 1개 회수
- 특정 칸 잠금 해제

11-3. 점수 및 랭킹
- 게임을 클리어하거나 종료했을 때 최종 점수를 계산한다.
- 점수를 이용해 랭킹을 보여줄 수 있다.

점수 요소 후보
- 도달한 층
- 처치한 몬스터 수
- 남은 체력
- 보유 골드
- 오버킬 피해량
- 퍼즐 판 완성 횟수
- 특정 모양 완성 횟수
- 클리어에 사용한 턴 수

11-4. 상태 이상 — 향후 우선순위 2
- 플레이어와 몬스터에게 일시적인 효과를 부여한다.

예시
- 화상
- 중독
- 취약
- 공격력 감소
- 실드 획득량 감소
- 특정 색상 봉인
- 퍼즐 판의 특정 칸 봉인

11-5. 추가 향후 기능
- 도감 등록 모양 자동 배치
- 특수 블록
- 복합 효과 블록
- 미믹 전투
- 공허 괴물 전투
- 광대 상자와 제거 불가능한 저주 블록
- 다회차 플레이와 보스 클리어 보상
- 아이템 구매


12. 추가 결정 사항

12-1. 지도와 진행
- 지도 생성 알고리즘
- 2층부터 9층까지 각 층의 노드 개수 범위
- 각 층의 전투 노드와 이벤트 노드 등장 비율
- 일반 몬스터와 엘리트 몬스터의 구분 여부
- 10층 보스의 구체적인 구성과 최종 클리어 연출

12-2. 덱과 블록
- 고정 기본 블록 12개의 I, O, L 종류별 개수
- 최초 선택 블록의 선택지 구성
- 덱의 최소 블록 수
- 일반 블록의 희귀도 적용 여부
- 현재 적용할 색상 종류와 색상별 능력
- 색상을 제거했을 때 무색 공격 10으로 돌아가는지 여부
- 복합 효과 블록의 구체적인 규칙

12-3. 퍼즐과 조작
- 플레이어 퍼즐 판 15칸의 정확한 초기 모양
- 체력이 5의 배수가 아닐 때 퍼즐 칸 계산 방식
- 체력 감소로 제거할 수 있는 칸 후보와 연결성 보장 방식
- 블록을 집지 않은 상태에서도 회전할 수 있는지
- 회수한 블록의 재배치가 배치 횟수를 다시 소비하는지
- 모바일 또는 터치 환경에서 회전하는 방식
- 배치 가능한 위치를 표시하는 정확한 시각 효과

12-4. 전투 규칙
- 약점의 정확한 위치, 표시 및 보너스
- 몬스터 퍼즐 판의 기본 기믹
- 특정 완성 모양의 종류와 기본 효과
- 완성 모양의 좌우 대칭, 높낮이 및 회전을 같은 모양으로 인정할지 여부
- 중복 세트 개수별 효과 배율
- 여러 세트가 같은 블록을 공유할 수 있는지
- 플레이어 퍼즐 판 완성 보너스의 최종 효과
- 색상별 공격, 방어, 회복 수치
- 몬스터별 체력, 공격력 및 행동 규칙
- 전투 포기 시 결과

12-5. 이벤트와 보상
- 일반 이벤트와 레어 이벤트 내부의 개별 등장 확률
- 상점의 블록 구매 가격과 판매 가격
- 유료 블록 추가의 최대 리롤 횟수
- 유료 블록 삭제의 최대 제거 횟수
- 미믹의 보물 상자 등장 확률
- 미믹과 공허 괴물의 난이도 및 보상
- 이벤트 전투에도 일반 전투 블록 선택 보상을 지급할지 여부
- 보스 전투의 기본 골드 보상 여부


13. 한 문장 요약

세계 제일의 대장장이가 무작위로 생성된 10층짜리 분기 던전에서 전투와 이벤트 노드를 선택하고, 매 턴 도구 주머니에서 블록 5개를 뽑아 최대 3개를 퍼즐 판에 배치하며 덱을 강화해 최종 보스를 처치하는 로그라이크 블록 배치 게임이다.


14. 명시되지 않거나 애매한 사항에 대한 Codex 재량 규칙

- 이 기획안에 명시되지 않았거나 표현이 애매한 세부사항은 Codex가 기존 기획 의도와 CODEX_DEVELOPMENT_GUIDE.md를 기준으로 결정할 수 있다.
- Codex는 구현을 진행하기 위해 반드시 필요한 세부사항만 결정한다.
- 명시된 수치, 핵심 게임 루프, 현재 적용 규칙 및 화면 흐름을 임의로 변경하지 않는다.
- '추후', '이후', '나중에', '향후 적용', '추가 개발', '후보'로 표시된 기능은 현재 구현하지 않는다.
- 향후 기능을 위한 확장 가능성은 고려할 수 있지만, 현재 코드에 완성되지 않은 기능이나 불필요한 복잡성을 미리 구현하지 않는다.
- 여러 선택지가 모두 가능하면 가장 단순하고 이해하기 쉬우며 현재 프로토타입 완성에 유리한 방식을 선택한다.
- UI의 미세한 위치, 간격, 색상, 애니메이션 시간, 내부 변수명처럼 핵심 규칙을 바꾸지 않는 사항은 Codex가 일관성 있게 결정할 수 있다.
- 미정 수치가 구현에 반드시 필요하면 조정하기 쉬운 상수 또는 데이터로 분리하고 임시 기본값을 사용할 수 있다.
- Codex가 재량으로 결정한 내용은 작업 완료 보고에서 '가정 또는 임시 결정'으로 명시한다.
- 선택에 따라 핵심 플레이 방식, 저장 데이터, 보상 구조 또는 대규모 파일 구조가 달라지는 경우에는 임의로 결정하지 않고 사용자에게 확인한다.
- 기획안과 개발 지침이 충돌하면 사용자의 최신 요청을 우선하며, 충돌 사실과 적용한 기준을 사용자에게 알린다.
